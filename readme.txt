                                         Инструкция по разворачиванию проекта
Используемые сокращения:
БД - база данных
СУБД - система управления базами данных
СКВ - система контроля версий

1) Установка зависимостей проекта.
   В консоли нужно перейти в папку проекта (это нужно делать для всех консольных команд, которые будут приведены
   в этой инструкции). После этого нужно ввести команду:

     +-------------------+
     | composer install  |
     +-------------------+

2) Нужно присвоить значение переменной окружения DATABASE_URL.
   Эта переменная должны содержать секретные и уникальные для каждой установки данные. Поэтому присваивать
   ей значение нужно в файле, который не может быть сохранён в системе контроля версий (Git). Рекомендуется
   использовать для этого файл .env.local. Формат д.б. следующий:

     +----------------------------------------------------------------------------------------------------------------+
     | DATABASE_URL="mysql://tenders_user:<пароль>@<имя_или_ip_хоста>:3306/tenders?serverVersion=8.2&charset=utf8mb4" |
     +----------------------------------------------------------------------------------------------------------------+

   Например, если пароль JKNFBEsf564jnf_DSfds, то установка значения переменной будет выглядеть так:

     +-------------------------------------------------------------------------------------------------------------------+
     | DATABASE_URL="mysql://tenders_user:JKNFBEsf564jnf_DSfds@MySQL-8.2:3306/tenders?serverVersion=8.2&charset=utf8mb4" |
     +-------------------------------------------------------------------------------------------------------------------+

   В файле .env, который может быть закоммичен в СКВ, переменной DATABASE_URL присвоено значение, которое содержит
   только несекретную часть данных, необходимых для подключения к БД. В этом обычно нет необходимости, и это значение
   будет просто перезаписано значением из .env.local. Это значение в .env нужно отчасти просто для дополнительного
   документирования, какая БД используется в проекте. Но оно также может стать необходимым, если переменная
   DATABASE_URL станет устанавливаться более сложным образом, чем в локальных .env-файлах. Тогда может возникнуть
   ситуация, что Doctrine для инициализации будет нужно знать версию СУБД раньше, чем она сможет получить значение
   DATABASE_URL вместе с секретными логином/паролем.

3) Создание пользователя СУБД и выдача ему прав на базу, которую планируется создать позже.
   Это можно сделать в любом клиенте MySQL (графическом или консольном) консольном клиенте MySQL.
   Пример SQL, где создаётся пользователь СУБД tenders_user с паролем JKNFBEsf564jnf_DSfds:

     +---------------------------------------------------------------------------------+
     | create user 'tenders_user'@'localhost' identified by 'JKNFBEsf564jnf_DSfds';    |
     | grant all on tenders.* to 'tenders_user'@'localhost';                           |
     +---------------------------------------------------------------------------------+

   Этот шаг важно выполнить перед созданием БД, если БД будет создаваться с помощью консольных инструментов Symfony,
   потому что Symfony будет использовать этого пользователя для подключения к СУБД и создания новой БД.

4) Создание БД.
   Есть несколько вариантов как это сделать:

      * Использовать инструменты Symfony и Doctrine. В этом случае в консоли нужно ввести команду:

          +-------------------------------------------+
          | php bin/console doctrine:database:create  |
          +-------------------------------------------+

        В этом случае будет создана БД c названием, которое было указано в DATABASE_URL.
        Поэтому в команде нет имени БД.

      * Использовать любой клиент MySQL. В этом случае, если использовать SQL, он будет таким:

          +---------------------------+
          | create database tenders;  |
          +---------------------------+

5) Применение миграций.
   Нужно ввести в консоли команду:

     +----------------------------------------------+
     | php bin/console doctrine:migrations:migrate  |
     +----------------------------------------------+

6) Загрузка фикстур. В этом проекте БД заполняется данными из .csv-файла с помощью фикстур.
   Нужно ввести в консоли команду:

     +--------------------------------------------------+
     | php bin/console doctrine:fixtures:load --append  |
     +--------------------------------------------------+
